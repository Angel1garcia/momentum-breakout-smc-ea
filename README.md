# Momentum Breakout + SMC Filter — Expert Advisor (MQL5)

> Automated trading system for MetaTrader 5, combining Smart Money Concepts (SMC) with momentum-based breakout detection and dynamic risk management. Designed for synthetic indices (Volatility 75, Crash/Boom) and Forex pairs.

## Overview

This Expert Advisor (EA) was built entirely in MQL5 and implements a multi-layer strategy that filters low-quality setups through a combination of trend, momentum, and market structure signals. It runs autonomously on MetaTrader 5, managing entries, exits, and risk without manual intervention.

The system was developed and refined through live trading experience in high-volatility environments, particularly on Deriv synthetic indices.

---

## Strategy Logic

The EA operates on a 3-layer confirmation model before opening any position:

### Layer 1 — Trend Filter (EMA)
- Uses a dual EMA system to define the prevailing trend direction
- Only allows long trades above the trend EMAs, short trades below
- Prevents counter-trend entries in strong directional markets

### Layer 2 — Momentum Confirmation (RSI)
- RSI filters out overbought/oversold entries in the direction of the trend
- Avoids entering exhausted moves, improving entry timing

### Layer 3 — Smart Money Concepts (SMC)
- **Break of Structure (BOS):** detects structural shifts in price that confirm institutional intent
- **Order Blocks (OB):** identifies supply/demand zones where institutional orders are likely resting
- **Fair Value Gaps (FVG):** detects imbalances in price delivery used as entry triggers or targets

A trade is only opened when all three layers align simultaneously.

## Risk Management

The EA includes a full risk management module designed to protect capital under adverse conditions:

| Feature | Description |
|---|---|
| **Percentage-based lot sizing** | Position size calculated as % of account balance per trade |
| **Dynamic Stop Loss** | Stop placed at structural invalidation level (below OB / BOS) |
| **Automatic Break-Even** | Moves SL to entry once price reaches a defined profit threshold |
| **Trailing Stop** | Locks in profits progressively as price moves in favor |
| **Daily Loss Limit** | EA stops trading for the day if max drawdown % is reached |
| **Daily Trade Limit** | Maximum number of trades per session to avoid overtrading |

## Live Dashboard

The EA includes a built-in live dashboard — no server, no Python, no external dependencies required.

**How it works:**
1. On every tick, the EA generates a complete self-contained HTML file (`dashboard_live.html`) with all trading data already embedded inside
2. The file is written directly to the MT5 `MQL5\Files\` folder
3. The browser opens this file locally and auto-refreshes every 5 seconds using a native HTML meta-refresh tag — no JavaScript fetch, no CORS issues, no security restrictions

This approach was specifically chosen to avoid browser `file://` security restrictions that block JavaScript from reading external files locally.

**Dashboard displays:**
- Symbol and EA status (Active / Waiting / Stopped)
- Real-time equity and balance
- Open position: direction, entry price, stop loss, take profit, lot size
- Open P&L (color-coded green/red)
- Daily trade count vs. limit
- Daily loss % vs. maximum allowed (with color-coded progress bar)
- EA settings summary and last update timestamp

## Setup & Installation

**Requirements:**
- MetaTrader 5 (any broker — optimized for Deriv)
- MQL5 compiler (included in MT5)
- Any modern web browser (Chrome, Edge, Firefox)

**Steps:**
1. Copy `MomentumBreakoutSMC.mq5` to your MT5 `Experts` folder
   - Default path: `C:\Users\[user]\AppData\Roaming\MetaQuotes\Terminal\[ID]\MQL5\Experts\`
2. Open MetaEditor (F4 in MT5) and compile with F7
3. Drag the EA onto your desired chart and configure input parameters
4. The EA will automatically generate `dashboard_live.html` inside `MQL5\Files\`
   - Go to **File → Open Data Folder → MQL5 → Files**
   - Open `dashboard_live.html` with your browser
   - The dashboard auto-refreshes every 5 seconds — no additional setup needed
